# Как запустить сервер

## Быстрый запуск

### Вариант 1: Обычный запуск
```bash
npm start
```

### Вариант 2: Запуск с автоперезагрузкой (для разработки)
```bash
npm run dev
```
*Сервер будет автоматически перезапускаться при изменении файлов*

### Вариант 3: Через готовый скрипт (Windows)
Просто дважды кликните на файл:
- `start-admin.bat` (для обычного запуска)

Или в PowerShell:
```powershell
.\start-admin.ps1
```

## После запуска

Сервер будет доступен по адресам:

- **Сайт:** http://localhost:3000
- **Админ-панель:** http://localhost:3000/admin
  - Логин: `admin`
  - Пароль: `admin123`
- **API Документация (Swagger):** http://localhost:3000/api-docs
- **Health Check:** http://localhost:3000/api/health

## Остановка сервера

Нажмите `Ctrl + C` в терминале, где запущен сервер.

## Проверка работы

1. Откройте браузер и перейдите на http://localhost:3000
2. Проверьте, что главная страница загружается
3. Проверьте админ-панель: http://localhost:3000/admin
4. Проверьте Swagger документацию: http://localhost:3000/api-docs

## Возможные проблемы

### Порт 3000 занят
Если порт 3000 уже занят, измените в файле `.env`:
```
PORT=3001
```
Или убейте процесс, занимающий порт:
```powershell
# Windows PowerShell
Get-NetTCPConnection -LocalPort 3000 | Select-Object -ExpandProperty OwningProcess | Stop-Process -Force
```

### Ошибка подключения к БД
База данных SQLite создается автоматически при первом запуске. Если есть проблемы, проверьте права доступа к папке проекта.

### Redis не подключен
Если Redis не установлен, сервер все равно будет работать, но без кэширования. Это нормально.
