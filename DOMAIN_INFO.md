# Информация о доменах

## Текущие домены

### Основной домен
- **Домен**: markinvesttour.com (или другой основной домен)
- **Статус**: Требуется уточнение у владельца

### Поддомены
- **afisha.neverend.travel** - используется в мета-тегах (требуется замена)
- **neverend.travel** - используется в ссылках на политику конфиденциальности (требуется замена)

## Что нужно сделать

### 1. Замена упоминаний neverend.travel
В коде найдены следующие упоминания домена neverend.travel, которые нужно заменить:

#### В файле `index.html`:
- Строка 1: `<meta property="og:url" content="https://afisha.neverend.travel" />`
- Строка 540: Ссылка на политику конфиденциальности `https://neverend.travel/policy`
- Строка 939: Ссылка на политику конфиденциальности в cookie-уведомлении

#### В других файлах:
- `afisha-australia/index.html` - аналогичные ссылки
- `australia-source.html` - аналогичные ссылки
- `australia-source-formatted.html` - аналогичные ссылки

### 2. Настройка DNS
После получения основного домена необходимо:

1. **Настроить A-запись** для основного домена, указывающую на IP-адрес сервера
2. **Настроить CNAME-запись** для поддомена www (если используется)
3. **Настроить SSL-сертификат** (Let's Encrypt или другой)

### 3. Обновление конфигурации сервера
В файле `server.js` и других конфигурационных файлах:
- Обновить CORS настройки с новым доменом
- Обновить URL в Swagger документации
- Обновить email-отправителя (если используется доменный email)

### 4. Проверка ссылок
После замены домена проверить:
- Все внутренние ссылки работают корректно
- API endpoints доступны
- Формы отправки работают
- Изображения и ресурсы загружаются

## Рекомендации

1. **Использовать переменные окружения** для доменов в коде
2. **Создать файл конфигурации** с настройками доменов
3. **Обновить robots.txt** с новым доменом
4. **Обновить sitemap.xml** с новыми URL
5. **Настроить редиректы** со старых доменов на новые (301 редирект)
