# Решение проблем с отображением сайта

## Основная проблема
В HTML файле остались артефакты от PowerShell скрипта обработки, из-за чего пути к CSS и JS файлам некорректны.

## Быстрое решение

### Шаг 1: Проверьте сервер
Убедитесь, что сервер запущен:
```powershell
npm start
```

### Шаг 2: Проверьте консоль браузера
1. Откройте сайт: http://localhost:3000
2. Нажмите F12 (DevTools)
3. Перейдите на вкладку "Console" или "Network"
4. Посмотрите, какие файлы не загружаются (ошибки 404)

### Шаг 3: Исправление путей

Если файлы не загружаются, выполните в PowerShell:

```powershell
# Замените все относительные пути на абсолютные
$content = Get-Content index.html -Raw
$content = $content -replace 'href="assets/', 'href="/assets/'
$content = $content -replace 'src="assets/', 'src="/assets/'
$content | Set-Content index.html -NoNewline
```

### Шаг 4: Удаление артефактов PowerShell

Если в HTML все еще есть артефакты типа `param($match)`, выполните:

```powershell
$content = Get-Content index.html -Raw
# Удаляем все артефакты PowerShell
$content = $content -replace '(?s)\s*param\(\$match\)[^"]*"assets/[^"]*"', ''
$content = $content -replace '(?s)\s*\$filename[^"]*"assets/[^"]*"', ''
$content | Set-Content index.html -NoNewline
```

## Альтернативное решение

Если проблемы сохраняются, лучше пересоздать HTML из оригинального файла:

1. Скопируйте `index_original.html` в `index.html`
2. Запустите правильный скрипт обработки
3. Или вручную замените все внешние URL на локальные пути

## Проверка работы

После исправления проверьте:

1. ✅ Все CSS файлы загружаются (проверьте Network tab)
2. ✅ Все JS файлы загружаются
3. ✅ Изображения отображаются
4. ✅ Анимации работают
5. ✅ Нет ошибок в консоли браузера

## Если ничего не помогает

Создайте новый HTML файл, скопировав структуру из `index_original.html` и заменив все пути вручную на `/assets/...`
