# Исправление проблем с отображением сайта

## Проблема
Сайт отображается некорректно из-за артефактов PowerShell скрипта в HTML файле.

## Решение

### Вариант 1: Использовать оригинальный HTML (рекомендуется)

Если проблемы сохраняются, лучше использовать оригинальный HTML файл и обработать его заново:

1. Скопируйте `index_original.html` в `index.html`
2. Запустите скрипт `process_html_complete.ps1` заново
3. Или вручную замените все пути в HTML

### Вариант 2: Ручное исправление

Откройте `index.html` в редакторе и найдите все вхождения:
- `param($match)`
- `$filename = [System.IO.Path]::GetFileName`
- `"assets/css/$filename"`
- `"assets/js/$filename"`

Замените их на правильные пути, например:
- `href="/assets/css/tilda-grid-3.0.min.css"`
- `src="/assets/js/jquery-1.10.2.min.js"`

### Вариант 3: Проверка через браузер

1. Откройте сайт в браузере
2. Нажмите F12 (открыть DevTools)
3. Перейдите на вкладку "Console" или "Network"
4. Проверьте, какие файлы не загружаются (будут показаны ошибки 404)
5. Исправьте пути к этим файлам в HTML

## Проверка работы сервера

Убедитесь, что:

1. Сервер запущен: `npm start`
2. Все файлы на месте в папках `assets/css/` и `assets/js/`
3. Пути в HTML начинаются с `/assets/` (абсолютные пути)
4. В консоли браузера нет ошибок загрузки ресурсов

## Быстрое исправление

Если нужно быстро исправить, выполните:

```powershell
# Замените все относительные пути на абсолютные
(Get-Content index.html) -replace 'href="assets/', 'href="/assets/' -replace 'src="assets/', 'src="/assets/' | Set-Content index.html
```

## Контакты

Если проблема не решается, проверьте:
- Логи сервера в консоли
- Консоль браузера (F12)
- Что все файлы существуют в папках assets
